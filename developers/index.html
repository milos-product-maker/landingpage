01 — CORE CONCEPT

What is an Auditor?

An Auditor is simply a standard OCI Container. You write your validation logic in Python, package it, and push it to the Lucid Marketplace.

Lucid injects your Auditor as a sidecar into the AI agent's Pod. This means zero code changes to your existing models. The Auditor intercepts requests and responses, applying your custom policies in real-time.

Infrastructure as Code

Define compliance policies in declarative manifests, version control them, and deploy with `kubectl apply`.

Familiar Tools

Use Python, Docker, and Kubernetes. No new languages or frameworks to learn.

sovereignty_auditor.py 
import lucid.auditor as auditor

# Pin execution to Swedish nodes
@auditor.on_request
def enforce_sovereignty(request):
    if request.node.country != "SE":
        return auditor.Deny("Sovereignty Breach")
    return auditor.Proceed()

# Redact sensitive data from responses
@auditor.on_response
def clean_output(response):
    if response.contains_pii():
        return auditor.Redact()
    return auditor.Proceed()
02 — BUILD & DEPLOY

Publish Your Own Auditor in Minutes

Step 1: Write Your Logic

Use our Python SDK to define request/response interceptors. The SDK provides helper methods for common tasks like PII detection, geo-fencing, and content filtering.

Step 2: Package as OCI Container

Standard Docker build. Your auditor runs as a lightweight sidecar container alongside the AI model.

Step 3: Push to Marketplace

Deploy to our registry. Your auditor is now available to be chained into any Lucid-powered AI deployment.

Dockerfile 
FROM lucid/auditor-base:latest

COPY sovereignty_auditor.py /app/auditor.py

ENTRYPOINT ["python", "/app/auditor.py"]
terminal 
# Build and push
$ docker build -t my-sovereignty-auditor .
$ lucid push my-sovereignty-auditor

✓ Auditor published to marketplace
03 — CHAIN AUDITORS

Compose Your Audit Chain

Declare your compliance policy in a simple YAML manifest. Mix and match pre-built auditors from the marketplace with your custom logic.

manifest.yaml 
apiVersion: lucid.sh/v1
kind: VerifiableJobManifest
metadata:
  name: finance-chain-v1
spec:
  auditors:
  # Developer Logic (Custom via SDK)
  - name: "dev-crash-catcher"
    image: internal/debugger:v1
  # Security Policy (MCP Compatible)
  - name: "sec-firewall"
    image: lucid-standard-v2:mcp-safe
  # Regulatory Logging
  - name: "gov-logger"
    image: gov-registry.eu/logger:v4
