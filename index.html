Example Auditor

sovereignty_auditor.py 
import lucid.auditor as auditor

# Pin execution to Swedish nodes
@auditor.on_request
def enforce_sovereignty(request):
    if request.node.country != "SE":
        return auditor.Deny("Sovereignty Breach")
    return auditor.Proceed()

# Redact sensitive data from responses
@auditor.on_response
def clean_output(response):
    if response.contains_pii():
        return auditor.Redact()
    return auditor.Proceed()

QUESTIONS

Frequently Asked Questions

Does this rely entirely on TEE security?

No. While we leverage hardware TEEs (Intel TDX/SGX, AMD SEV) for the strongest isolation guarantees, our verification layer provides additional cryptographic attestation. We are transparent about hardware dependencies and recommend defense-in-depth strategies.

Can I use this on AWS, Google Cloud, or Azure?

Yes. Lucid supports multi-cloud deployments. We work with confidential computing instances available on major hyperscalers (AWS Nitro Enclaves, Azure DCsv3, GCP Confidential VMs) as well as on-premise and sovereign cloud providers.

Who is the actual customer?

Lucid serves both technical and business stakeholders. While developers install and integrate the platform, the Compliance Officers, CROs, and General Counsels are the economic buyers who benefit from reduced audit burden and regulatory risk.

What happens if I'm already using model guardrails?

Lucid complements existing guardrails. Our Auditor Chains wrap around your existing safety measures, adding cryptographic attestation and audit trails. We don't replace your safety stack—we make it provable.
